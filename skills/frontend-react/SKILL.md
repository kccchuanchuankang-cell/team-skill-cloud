# Frontend React

## When To Use

Use this skill for:

- React component work
- Screen or form implementation
- Client-side state and interaction flow changes
- UI refactors that may affect usability
- Styling work that should follow team conventions

## Workflow

1. Identify the route, page, or component entry point.
2. Read [references/ui-rules.md](references/ui-rules.md) before making layout or interaction changes.
3. Preserve established design patterns within the consuming project.
4. Make loading, empty, error, and success states explicit where relevant.
5. Validate both desktop and mobile impact if the project is responsive.

## Validation

- Check state transitions and form behavior
- Check accessibility basics for controls and labels
- Check visual consistency with the surrounding project
- Check for missing edge states
