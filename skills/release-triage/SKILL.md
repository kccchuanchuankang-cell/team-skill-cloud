# Release Triage

## When To Use

Use this skill for:

- Pre-release checks
- Regression-oriented code review
- Incident or issue triage
- Release notes sanity checks
- Risk-focused review of recent changes

## Workflow

1. Identify the target release scope or change set.
2. Read [references/release-process.md](references/release-process.md).
3. Prioritize behavior regressions, missing tests, and rollback risk.
4. Report findings in severity order.
5. Call out testing gaps explicitly when confidence is limited.

## Output Expectations

- Findings first
- Open questions second
- Summary last
