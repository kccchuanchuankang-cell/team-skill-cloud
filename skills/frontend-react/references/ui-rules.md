# UI Rules

Use this reference when changing React UI.

## Guardrails

- Match the surrounding design system before inventing a new pattern
- Prefer clear hierarchy and readable spacing over dense layouts
- Make empty, error, and loading states visible
- Avoid adding heavy abstraction unless the project already uses it

## Review Questions

1. Does the new UI match nearby screens?
2. Is the primary action obvious?
3. What happens when data is empty, slow, or invalid?
4. Are labels and affordances clear enough for first-time users?
